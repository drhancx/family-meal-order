# 🍳 "今天想吃什么" — 家庭点菜 Web App 开发提示词

---

## 项目背景

请帮我开发一个 **家庭点菜 H5 Web App**，这是我送给爱人的礼物。只有我们两个人使用：一个人点菜（点菜者），另一个人接收点菜内容（接单者/大厨）。不需要走微信小程序审核，做成手机网页即可，可以在微信中直接打开链接使用。

---

## 技术要求

- **框架**：React 单页应用（Vite 构建）
- **样式**：Tailwind CSS
- **部署目标**：Vercel 或 GitHub Pages（纯前端，免费托管）
- **数据存储**：浏览器 localStorage（两人使用，无需后端数据库）
- **通知机制**：点菜完成后生成一段文字摘要，可一键复制发送到微信；或者通过共享链接让对方查看订单
- **适配**：以手机竖屏为主，适配 iPhone 和安卓主流尺寸，在微信内置浏览器中完美运行

---

## 页面结构与功能

### 1. 首页（欢迎页）
- 温馨的欢迎语，例如 "亲爱的，今天想吃什么？"
- 可以放置我上传的照片作为背景或装饰元素
- 两个入口按钮：
  - 🍽️ "我要点菜"（进入点菜页面）
  - 👨‍🍳 "查看订单"（进入接单/查看页面）

### 2. 点菜页面
- **顶部**：横向可滑动的菜系分类标签（如：凉菜、热菜、汤羹、主食、甜品/饮品）
- **菜品卡片列表**：
  - 每道菜一个卡片，包含：菜品图片（占位图或 emoji 占位）、菜名、简短描述、"加入" 按钮
  - 点击 "加入" 后按钮变为数量加减控件（➖ 1 ➕）
  - 卡片布局：每行2个，网格排列
- **底部悬浮栏**：
  - 显示已选菜品数量和简要列表
  - "提交今日菜单" 按钮
- **提交后**：
  - 弹出温馨提示（如 "已告诉大厨啦~ 💕"）
  - 生成订单摘要文字，可一键复制到微信发送给对方
  - 同时将订单保存到 localStorage

### 3. 订单查看页面
- 显示最近提交的点菜订单
- 内容包括：点菜时间、菜品列表及数量
- 简洁的卡片式展示
- 支持查看历史订单（按日期分组）

### 4. 菜品管理页面（可选/隐藏入口）
- 长按首页标题或特定手势进入
- 可以增删改菜品：菜名、分类、描述、图片
- 方便我后续自己维护菜单内容

## UI 设计风格

### 整体氛围
- **温馨、浪漫、居家感**，像在家里的厨房一样温暖
- 色调以 **暖色系** 为主：奶油白底色 + 暖橘/珊瑚色点缀 + 少量柔粉色
- 圆角卡片、柔和阴影，整体圆润可爱
- 适当使用 emoji 增加生活气息 🍳🥘💕

### 字体
- 中文使用系统默认（-apple-system, "PingFang SC", "Microsoft YaHei"）
- 标题可以稍大且加粗，营造亲切感
- 欢迎语可以用手写风格字体（如 Google Fonts 的 "Ma Shan Zheng" 或 "ZCOOL XiaoWei"）

### 动效
- 页面切换使用平滑过渡
- 菜品加入购物车时有轻微弹跳动画
- 提交订单时有撒花或爱心飘落效果 💕
- 按钮点击有涟漪反馈

### 特殊元素
- 首页可以放我们的照片（预留图片位置，使用占位图，我后续替换）
- 页面底部可以有一句小情话，随机展示（如 "做你爱吃的，是我最开心的事 ❤️"）

## 关键交互细节

1. **点菜流程要丝滑**：点击加入 → 底部栏实时更新 → 提交 → 生成摘要 → 一键复制
2. **订单摘要格式示例**：
   ```
   📋 今日点菜单
   ──────────
   🥒 凉拌黄瓜 × 1
   🍅 番茄炒蛋 × 1
   🍖 红烧肉 × 1
   🍚 米饭 × 2
   ──────────
   共 4 道菜
   期待大厨的手艺~ 💕
   ```
3. **复制成功后提示** "已复制到剪贴板，快去微信发给 TA 吧~"
4. **空状态处理**：没有点菜时显示可爱的提示，如 "还没选菜呢，快翻翻菜单吧 🍽️"

```
src/
├── components/
│   ├── WelcomePage.jsx       # 首页
│   ├── MenuPage.jsx          # 点菜页面

│   └── dishes.js             # 预置菜品数据
├── hooks/
│   └── useCart.js             # 购物车状态管理
├── utils/
│   └── orderUtils.js         # 订单生成、复制等工具函数
├── App.jsx
├── index.css
└── main.jsx
```

5. 开发订单提交和复制功能
6. 开发订单查看页面
7. 添加动效和细节打磨
8. 适配手机端样式
9. 部署到 Vercel
